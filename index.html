I built a custom TradingView indicator with Claude Code & Fable 5.

It's called the Storm Gauge and is built off a real quant trading strategy.

I open-sourced the full code on GitHub. Free to install, free to fork, yours to improve.

Here's how to install a quant indicator on your TradingView chart:

What it actually is

The Storm Gauge is a live implementation of the GARCH model, a Nobel Prize-winning volatility framework that real quant desks run daily.

It forecasts how "violent" tomorrow's market could be by combining three inputs: an asset's baseline volatility, yesterday's shock, and where volatility was already sitting before that shock happened.

It doesn't predict market direction. Instead, it measures risk, in real time, on your actual chart.

How to install it

Method 1. Plugin command

Open the GitHub repo: https://t.co/E1evtqgKJp

Find the installation section, copy the command, and paste it into Claude Code. It runs the plugin install automatically.

Method 2. Manual config

Open garchmethod.md in the repo, copy the entire file, and paste it into Claude Code. It fetches the skill files directly and verifies the strategy for you.

(you only need one method; I'm just showing both)

Getting it onto your TradingView chart

Inside the repo, there's a Pine Script folder. Open it, copy the entire file.

Go into TradingView's Pine Editor, paste it in, hit Enter, and refresh.

That's it. The Storm Gauge now runs live on your chart as a real number.

Once it's installed, just talk to it:

→ "What's the volatility forecast on Bitcoin?"
→ "Explain what the current volatility forecast means on $BTC and how it should impact my position sizing"
→ "Help me size my S&P500 position according to current market volatility"

Does it actually work?

I backtested the same EMA cross strategy two ways across 15 years of BTC data. Same entries, same exits.

→ Fixed position sizing: $17,957 final equity
→ Storm Gauge (GARCH) sizing: $21,205 final equity

Fewer drawdowns, less risk, better result.

谷歌财报炸裂成这个样子，为什么AI股还是跌了

本来说熬穿等财报的，没完全熬住。
熬到一点多睡觉了，四点钟起床瞄了一下，五点钟又起床瞄了一下，然后继续睡着了。

醒来重新把谷歌和特斯拉的财报看了一遍，让我们一起看看昨晚两家巨头交出的答卷：

谷歌交出了一份很好的财报，但市场没有因此变得更乐观
特斯拉交出了一份不算差的收入答卷，却再次暴露了利润和现金流的问题

昨天财报前，我觉得会有两个剧本

第一，谷歌财报真的很好，市场确认AI逻辑，随后迎来更大的反弹
第二，财报没有那么好，主力提前拉高，借着散户对财报的期待完成出货

结果出来以后，感觉两种剧本同时发生：谷歌交出了第一种剧本的基本面，却走出了第二种剧本的价格反应。

就像我经常跟社群群友讲的，没人能预判未来。

谷歌本季度营收1198亿美元，同比增长24%；搜索业务增长17%，YouTube广告增长13%。最关键的Google Cloud收入达到247.7亿美元，同比增长82%，营业利润也从去年同期的28.3亿美元增长至88.1亿美元。

这次，终于不再是【AI 以后可能赚钱】了。

AI基础设施、企业AI服务和TPU，正在真实推动谷歌云的收入与利润增长。
谷歌公布的净利润高达1121亿美元、每股收益9.11美元，不过这个数字不能直接和市场预期比较，因为其中包含约990亿美元的股票投资收益。

扣除这部分影响以后，谷歌的主营业务依旧很强，但没有账面利润看起来那么夸张。

让市场害怕的，是资本开支。

谷歌第二季度资本开支达到449亿美元，首次出现约59亿美元的负自由现金流；同时将今年资本开支指引从1800亿至1900亿美元，提高到1950亿至2050亿美元。
电话会后，谷歌盘后一度跌超3%。
老生常谈了，市场已经不再满足于听“AI需求很好”。

现在大家开始继续追问更多的东西，毕竟，欲望是一条无穷无尽的沟壑。

所以谷歌这份财报，对谷歌股价未必是纯利好，但对AI基础设施产业链是一次非常强的需求验证。

老马的 $TSLA 更复杂

二季度营收282.4亿美元，同比增长26%，汽车收入增长23%，能源业务增长13%，服务业务增长50%。车辆交付达到48.01万辆，储能部署达到13.5GWh，说明销量和业务规模确实在恢复。
但特斯拉的营业利润只有3.98亿美元，同比下降57%，营业利润率降到1.4%；调整后每股收益0.33美元，低于市场预期。
资本开支同比增长142%至57.9亿美元，自由现金流转为负10.9亿美元。

特斯拉现在的问题不是没有故事。
Robotaxi、Optimus、Cybercab、AI训练集群和自建半导体能力，每一个故事都大的吓人。

问题是这些故事正在进入最烧钱的阶段，而汽车业务暂时还没有提供足够厚的利润垫。
财报之后，特斯拉盘后跌幅一度超过3%。
所以昨晚的两份财报放在一起看，给出的信号其实很清楚：AI需求没有结束，但市场开始认真计算AI的成本。

这也是我对后市的基本判断。
昨晚的财报不足以直接证明纳斯达克会继续大涨，AI行情也没结束。

所以接下来，不会再出现一起科技股涨的场面，板块内部会继续分化：有真实订单、收入和利润兑现的公司，会获得更高容忍度。
只有资本开支、没有商业回报的公司，会被市场重新审视。
而帮助巨头建设算力、网络和数据中心的上游公司，反而会继续获得订单。

我没有直接去赌谷歌财报，而是提前在 195埋伏了一些 $NBIS ，192埋伏了一些 $MRVL 。

目前收益都还不错。
其实 $MRVL 我自己喊的位置是170-180，不过自己也没能拿到就是了。

NBIS所在的Neocloud板块，核心逻辑就是为AI公司提供GPU集群、训练和推理基础设施。

谷歌云收入增长82%、仍然需要继续加大数据中心投资，说明市场对AI算力的需求并没有被证伪。
Nebius本身也将业务定位为面向AI开发者的全栈云基础设施。
当然，谷歌的负自由现金流也提醒我们，Neocloud同样是一个非常重资产的行业。
需求很大，不代表每家公司都能赚钱。
后面还要继续看NBIS的客户质量、利用率、融资能力和算力扩张效率。

MRVL的逻辑则更加偏向“连接”。

GPU越来越多，集群越来越大，真正的瓶颈会从单颗芯片，逐渐转移到芯片之间、机架之间和数据中心之间的通信。
Marvell覆盖光学DSP、交换芯片、硅光、定制ASIC和数据中心互联。谷歌继续扩大TPU和AI数据中心，不代表MRVL一定拿到了谷歌的某一笔订单，但可以确认，它所在的高速互联和定制芯片赛道仍处于需求扩张阶段。

所以接下来，我最关注的不是谷歌和特斯拉今天究竟跌几个点，巨头的股价波动不会太大。

而是谷歌下跌以后，AI云、光通信和数据中心基础设施能不能继续获得承接。

假如谷歌因为资本开支和现金流下跌，但NBIS、MRVL以及其他AI基础设施公司表现相对更强，说明市场正在把谷歌的支出，理解成上游公司的收入。

反过来，如果整条AI产业链一起下跌，那就说明财报前的反弹确实打入了太多乐观预期，后面还需要继续消化。

财报季最容易犯的错误，就是把“财报很好”和“股票一定上涨”画上等号。

公司决定长期价值
预期和买入成本，决定短期价格

财务数据公布后，股价最初只是震荡，大致没有明确方向。直到CFO在电话会上把全年资本开支指引提高150亿美元，并表示2027年支出还会显著增长，股价才开始明显走低。

这说明市场并没有否定Cloud的增长。

市场只是认为，Cloud超预期带来的价值，还不足以立刻覆盖更高资本开支、更低自由现金流和未来折旧增加带来的影响。

财报后，卖方模型很可能出现两种相反的调整。

一边上调Cloud收入、Cloud利润和长期收入预测。

另一边下调自由现金流、股票回购，增加未来资本开支、折旧和利息费用。

这两种变化完全可以同时发生。

公司长期收入天花板提高了，不代表股票短期估值一定提高。

你看对了产业趋势，也不代表市场必须在第二天给你奖励。

对于散户来说，最重要的是把公司判断和仓位判断分开。

公司判断上，这份财报明显增强了谷歌的长期逻辑。

Search没有出现明显失速，AI Mode正在增加查询量，单次回答成本持续下降。
Cloud需求不仅来自合同，也体现在客户实际使用量超过承诺。
Cloud利润率还大幅提高。

但仓位判断上，仍然有几个问题没有答案。

Cloud增长82%里，有多少来自长期可持续的软件和服务收入，又有多少来自TPU系统销售和短期算力紧缺？

Cloud利润率35.6%能否维持，还是会随着第三方算力采购、硬件收入占比增加而回落？

资本开支在提高到2000亿美元左右之后，2027年究竟还会提高多少？

自由现金流何时能够重新覆盖资本开支？

搜索收入增长17%，是AI真正改善了产品，还是也受到了世界杯等事件性广告和查询需求的帮助？

这些问题不影响你长期看好谷歌。
但它们会影响你应该在什么价格、用多大的仓位表达这种看好。

我更认可的做法，是把加仓拆成几笔。

第一笔，买这份财报已经确认的事实，搜索韧性仍在，Cloud已经成为第二利润中心。

第二笔，等待市场消化资本开支和分析师盈利预测调整，观察股价能否在坏消息充分讨论以后企稳。

第三笔，等待下一季度验证：Cloud在高基数下是否仍能保持50%以上增长，利润率是否维持30%以上，搜索是否继续保持中双位数增长，资本开支是否不再大幅超出市场预期。

不要特意追求完美买点。

毕竟，好公司和好交易之间，中间还隔着估值、预期和仓位。

总结下来，谷歌这次财报就一句话概括：

公司比预期更强，市场担心的事情也比预期更严重。
